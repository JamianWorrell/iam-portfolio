Role Mining

Analyze identity entitlement patterns to discover candidate roles. PointSail examines access shared across identities in the same department and suggests business roles you can create to streamline provisioning.

🔐 Identity Lifecycle

Every identity follows a Joiner → Mover → Leaver pattern. PointSail automates provisioning at each stage using lifecycle rules triggered by authoritative source events (e.g. Workday hire).

Joiner: New hire — provision birthright access based on role/dept.
Mover: Transfer — adjust access for new role, revoke old.
Leaver: Termination — immediately disable all access, start offboarding SLA.
⚖️ Separation of Duty

SoD prevents a single person from having conflicting access that enables fraud. Example: someone who can create a vendor in SAP should not also be able to approve payments to that vendor.

Left-hand population: Set of entitlements/roles (e.g. SAP Finance Write).
Right-hand population: Conflicting set (e.g. SAP Vendor Admin).
Exception: Business-justified time-limited bypass with CISO approval.
📋 Access Certification

Periodic review where managers or entitlement owners certify that each identity still needs their current access. Revoked items trigger automatic de-provisioning.

Campaign types: Manager Review, Entitlement Owner, Identity, Application.
Escalation: If no decision by deadline, escalate to backup certifier.
Best practice: Quarterly for privileged, annual for standard access.
🏗️ Role Engineering

Roles bundle entitlements so provisioning is policy-driven, not ad-hoc. Role mining discovers patterns in existing access to suggest roles you should create.

Business Role: Maps to a job function (e.g. Finance Analyst).
IT Role: Technical/privileged access (e.g. AD Domain Admins).
Entitlement: A specific permission in an application (atomic unit).
